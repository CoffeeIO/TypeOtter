When rendering the controls bar will have a print option, in the Chrome browser there will be an option to save as pdf which saves the rendered document... as a pdf. This works pretty well, however I would recommend using Adobe Reader when displaying the pdf for the most accurate display.
Any document can also be saved as html, this is only really useful when create documents with an online editor and you want to save html, css and js in a single file quickly. This will not download the content of external script or images, so you still need a internet connection for that.
While mobile support is not the biggest focus of the project, I still think it's important to allow any as many users to create their documents. Currently full support only covers Android platform, as it's required to download files and like on the desktop, only Chrome has the save as pdf option.
